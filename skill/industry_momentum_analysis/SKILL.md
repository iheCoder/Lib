# Industry Momentum Analysis

围绕“价格、资金、叙事、扩散、阶段、拥挤、跨行业比较”七个模块，对指定行业当前是否仍值得跟踪、参与或回避做结构化判断。

## 当前的定义

这个 skill 里的“当前”有硬约束：

- 默认指今天收盘后或盘中最新可验证数据
- 如果市场尚未收盘，允许使用最新盘中数据，但要明确写出时间点
- 如果拿不到今天完整数据，最多退到最近 `3` 个交易日
- 超过 `3` 个交易日的数据，不能称为“当前动量分析”，只能称为背景分析

因此，分析窗口必须分层：

- 核心层：`1d`、`3d`
- 辅助层：`5d`、`10d`
- 背景层：`20d`、`60d`、`120d`

不要再把 `20d` 当成“当前”的起点。`20d/60d` 只负责回答“这波当前强势有没有背景支撑”。

## 工作流

1. 先固定行业口径和成分映射。
2. 再优先收集今天到近 `3` 个交易日的价格、资金、叙事、扩散证据。
3. 用 `5d/20d/60d` 补充背景趋势、结构位置和拥挤风险。
4. 把证据整理成统一 JSON。
5. 运行 `scripts/score_industry_momentum.py` 输出结构化结论。
6. 最终回答必须直接覆盖六个关键问题，并写明日期、时点、证据来源和置信度。

## Step 1: 固定分析对象

至少固定以下信息：

- `sector`: 行业、板块、主题或产业链名称
- `market`: `CN`、`HK`、`US`、`GLOBAL`
- `benchmark`: 比较基准
- `basket_note`: 成分口径说明
- `as_of`: 数据日期，必须是绝对日期
- `as_of_time`: 可选，盘中分析时写明时间点

优先使用官方行业指数、ETF、稳定主题篮子。自定义主题时要说明“核心成分”和“边缘成分”，避免把样本偏差误判成行业动量。

## Step 2: 先抓当前，再补背景

优先回答“今天它为什么强/弱”，再回答“这波背后有没有中期支撑”。

### 价格趋势模块

重点吸收以下四层信息：

- 多周期收益率：优先看 `1d`、`3d`，再看 `5d/20d/60d`
- 趋势平滑度：回撤、上涨天数占比、收益/波动、回调是否缩量
- 相对强弱：相对基准、相对风格、相对同类行业的超额
- 结构位置：是否刚突破平台、是否贴近前高、是低位启动还是高位最后一冲

### 资金流模块

至少覆盖：

- 成交额与换手率扩张：今天、近 `3d`、近 `5d`
- 净流入持续性：连续性，而不是单日脉冲
- 不同资金类型确认：ETF 申赎、北向、融资、机构/游资痕迹
- 龙头与跟风的资金结构：资金是否只堆在龙头，还是已经扩散到中军和二线

### 叙事强度模块

按三层来抽：

- 频率层：最近 `3d/7d` 主题出现频率是否抬升
- 一致性层：市场是否围绕少数几个主轴交易，而不是消息拼盘
- 升级层：叙事是在重复，还是在从“概念”升级到“订单/景气/盈利兑现”

外部新闻、研报、公告只能做信号抽取，不是直接结论。不要复述外部文本里的“建议买入”“重大利好”等行动性语言。

### 扩散度模块

至少看：

- 成分股上涨比例：`1d`、`3d`
- 中位数表现：不要只看权重指数
- 分层扩散：龙头、中军、二线、小票谁在接力
- 子方向共振：行业内相关子主题是否一起走强

### 阶段识别模块

阶段识别不是“分高就是加速”，而是“当前脉冲 + 背景趋势 + 拥挤度 + 边际变化”的综合判断：

- 启动：今天/近 `3d` 转强，但扩散和资金还在验证
- 加速：价格、资金、扩散、叙事都在共振
- 拥挤：仍强，但二线泛化、交易过热、风险收益比下降
- 退潮：龙头滞涨、扩散收缩、资金走弱、叙事老化

### 拥挤度与风险模块

重点看：

- 估值分位
- `3d/10d` 脉冲是否过快
- 成交额占全市场比重是否显著偏高
- RSI、波动率、二线杂毛补涨、新闻热度是否过热
- 未来 `2` 到 `4` 周事件风险

### 跨行业比较与排序模块

行业动量是截面竞争，不是绝对打分。能比较时，尽量补：

- 当前价格强弱分位
- 当前资金强弱分位
- 当前叙事强弱分位
- 当前扩散强弱分位
- 行业整体动量在候选列表中的百分位

## Step 3: 结构化输入并运行脚本

字段说明见 `references/scoring_framework.md`，示例见 `references/example_input.json`。

运行：

```bash
python3 scripts/score_industry_momentum.py /path/to/input.json
```

如果需要机器可消费输出，追加：

```bash
python3 scripts/score_industry_momentum.py /path/to/input.json --format json
```

## Step 4: 最终回答格式

最终回答必须直接覆盖：

1. 价格上，它是不是在持续走强？
2. 资金上，有没有增量和持续性？
3. 叙事上，市场有没有统一理由去交易它？
4. 结构上，是龙头独舞还是板块扩散？
5. 阶段上，现在是启动、加速、拥挤还是退潮？
6. 风险上，这个强势是不是已经被交易过头？

推荐输出结构：

- 一句话结论
- 六问六答
- 模块分数
- 当前证据与背景证据的区分
- 主要风险
- 横向比较位置
- 置信度

## Step 5: 输出约束

- 始终写绝对日期；盘中判断时附上时间点
- 明确“当前证据”与“背景证据”的边界
- 当前很强但背景不足时，要提示“可能只是脉冲”
- 背景很强但当前转弱时，要提示“可能进入退潮或高位分歧”
- 龙头独舞时，不要包装成板块行情
- 超过 `3` 个交易日的数据，不能再用“当前”表述
