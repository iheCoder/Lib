# 浏览器站点技能

这个 skill 提供稳定的网页自动化能力，核心是用**多模态目标描述**替代脆弱的 CSS/XPath selector，并把一次探索成功**沉淀为可复用的站点技能**。

## 何时使用

- 用户说：**浏览器操作**、网页自动化、站点工作流、页面交互
- 用户说：**沉淀站点技能**、站点 skill、workflow spec、编译站点知识
- 用户说：**多模态目标**、语义定位、稳定锚点、页面状态验证
- 用户要求自动化某个网站的特定流程

## 核心规则（必须严格遵守）

### 禁止行为
- ❌ **严禁只用单一 selector 描述目标**（如 `#submit-btn` 或 `.checkout-button`）
- ❌ 严禁在页面变化后仍沿用过期锚点
- ❌ 严禁点击/填写后不验证状态变化
- ❌ 严禁把一次性探索直接固化为站点技能（必须先抽象、参数化）

### 必须行为
- ✅ **每个可操作目标必须用多模态描述**（语义 + 文本 + role + DOM 路径 + 视觉线索 + 前后状态）
- ✅ 每次关键动作后立刻验证 `state_after`，失败则视为未完成
- ✅ 当锚点冲突、页面漂移或状态不满足时，必须重新观察
- ✅ 沉淀站点技能前必须区分"稳定部分"与"变化项"

## 两种工作模式

### 模式 1：探索并执行 (explore-and-act)

用户要求在某个网站完成具体任务时使用。

**执行流程：**
1. **理解目标**：明确用户要完成的任务与期望状态
2. **多模态观察**：打开页面，识别关键元素，构建多模态目标描述
3. **执行动作**：基于多模态锚点执行操作（点击、填写、导航）
4. **验证状态**：每次动作后立刻验证 `state_after`
5. **记录轨迹**：保存关键页面、目标描述、动作链、分支决策

**输出：**
- 任务完成结果
- 探索轨迹文档（可选，为后续沉淀做准备）

### 模式 2：沉淀站点技能 (crystallize-site-skill)

用户要求把一次成功探索编译成可复用站点技能时使用。

**执行流程：**
1. **加载轨迹**：读取探索模式产生的轨迹文档
2. **抽象分析**：识别稳定部分（站点结构、关键目标）与变化项（搜索词、日期、用户输入）
3. **参数化**：把变化项提取为参数，保留稳定锚点
4. **编译模板**：基于 `site-skill-template.md` 生成站点技能文档
5. **用户确认**：展示站点技能草稿，收集反馈
6. **写入文件**：确认后写入独立站点技能文件

**输出：**
- 站点技能文档（可被后续调用复用）

## 快速开始

### 场景 1：自动化某个网站任务

用户：「帮我在某购物网站搜索 iPhone，筛选价格 5000-8000 的商品，加入购物车」

你应该：
1. 进入探索模式
2. 访问网站，用多模态描述识别搜索框：
   ```json
   {
     "semantic_name": "product_search_input",
     "visible_text": ["搜索", "商品名称"],
     "aria_role": "searchbox",
     "dom_path_summary": "header > search section",
     "state_before": ["页面已加载", "搜索框可见"],
     "state_after": ["输入框包含 'iPhone'"]
   }
   ```
3. 执行填写，验证状态
4. 继续后续步骤，记录完整轨迹

### 场景 2：沉淀站点技能

用户：「把刚才的探索沉淀成可复用的站点技能」

你应该：
1. 进入沉淀模式
2. 分析轨迹，识别参数：`search_keyword`、`price_range`
3. 保留稳定目标：搜索框、价格筛选器、加购按钮
4. 基于模板生成站点技能文档
5. 展示给用户确认

## 失败处理

### 目标定位失败
- **现象**：找不到匹配的元素，或找到多个候选
- **处理**：重新观察页面，补充视觉线索或 DOM 路径细节，缩小候选范围

### 状态验证失败
- **现象**：动作执行后 `state_after` 未满足
- **处理**：检查是否有弹窗、加载延迟、错误提示；必要时重试或调整策略

### 页面漂移
- **现象**：站点改版，原有锚点失效
- **处理**：重新探索，更新多模态描述；如果是已沉淀的站点技能，标记为待更新

## 工具路径

本 skill 主要指导 Agent 的思考与执行方式，不依赖特定脚本。可配合使用的工具：
- 浏览器工具（如 Playwright、Puppeteer、Selenium）
- 视觉识别工具（截图 + OCR/目标检测）
- DOM 分析工具（XPath、CSS Selector、可访问性树）

## 与其他 skill 的关系

- **vs scrapling**：scrapling 专注网页抓取（获取内容），本 skill 专注网页操作（交互、填表、点击）
- **vs open-prose**：可配合使用，open-prose 负责工作流编排，本 skill 负责单站点自动化的稳定执行
